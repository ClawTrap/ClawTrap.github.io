Abstract

Why ClawTrap matters

Current evaluations of autonomous GUI agents, such as OpenClaw, often rely on sterile simulated environments or text-based toy tasks. In deployment, however, users grant these agents high-level privileges to operate applications autonomously. This creates a critical security question: what happens when the agent encounters malicious visual interventions during execution?

The Core Motivation

ClawTrap assumes a realistic threat model in which the agent already has meaningful privileges and an attacker performs man-in-the-middle visual attacks, such as fake pop-ups, manipulated UI elements, or poisoned messages, to hijack the agent's logic, expose sensitive data, or induce system failure.

We introduce ClawTrap, a highly configurable arena that injects dynamic multi-modal adversarial threats into daily computing tasks including communication, web browsing, and shopping. Our evaluation shows that state-of-the-art GUI agents remain strikingly vulnerable to visual spoofing and malicious instructions embedded in ordinary interfaces.

Pop-up spoofing attack

A believable intervention, timed during legitimate navigation.

While the OpenClaw agent is navigating a legitimate site, an attacker inserts a realistic two-factor authentication overlay that visually dominates the task interface. The page still appears ordinary, but the agent's attention is redirected to a malicious prompt.

  • Interrupts task execution with a security-pretext modal that appears routine.
  • Can trigger secret retrieval or unsafe local-file access if the agent trusts the popup.
  • Turns ordinary browsing into a data exfiltration or behavior-hijacking channel.

Taxonomy

Attack families studied in ClawTrap

Attack Family 01

Dynamic Pop-up Attacks

Inject deceptive UI layers that hijack the agent's execution flow at exactly the wrong moment.

  • Privacy Harvesting: fake authentication windows request local secrets.
  • Decision Hijacking: deceptive confirm buttons redirect behavior.
  • DoS Prompts: ultra-long text stresses or crashes the agent context.

Attack Family 02

Contextual Poisoning

Embed malicious instructions inside otherwise normal communication and information channels.

  • Social Media: prompts instruct the agent to send money or reveal data.
  • Email Hijacking: replies are manipulated to leak recent correspondence.
  • Identity Spoofing: messages impersonate the owner or a trusted authority.

Attack Family 03

Real-World Scenario Traps

Blend attacks directly into ordinary GUI tasks so that malicious intent is hard to distinguish.

  • E-commerce: fake payment buttons redirect funds during checkout.
  • Information Retrieval: altered results induce hallucination and bad actions.
  • Local Learning: poisoned files distort memory or future planning behavior.
